# NetForge — Planering: kärnstabilitet innan nyutveckling
**Gäller från:** 2026-06-10  
**Mål:** Få kärnan (topologi, progress, inlärning) felfri och production-ready. Login, notifieringar och sociala funktioner ligger *utanför* scope tills core är grönt i `tsc --noEmit` + `npm run build`.

---

## 1. Principer
- **Core först.** Inga nya features (t.ex. `/login`, profiler, achievements) förrän topologi-generering, progress-lagring och a11y är stabila.
- **Verifiera med verktyg.** Varje steg verifieras med `tsc --noEmit` och vid större ändringar `npm run build`. Ingen kod läggs i `main` utan grönt bygg.
- **Agent-upplägg:** Varje område kan köras i en egen agent-omgång (parallellt eller sekvensiellt beroende på beroenden). Beroende-kedja: topologi → progress → UI/a11y → test.

## 3. Del B — Förbättringar (core-förbättringar, kräver stabil buggfix)

### 3.1 Topologi
- **T1. Validator för alla genererade topologier**  
  Skriv `lib/topology/validate.ts` med funktioner:
  - `validateNoIpConflict(devices)`
  - `validateNoOverlap(devices, links)`
  - `validateLinks(devices, links)`
  - `validateSubnets(devices)`
  
  Kalla validatorn i `generateByKey()` och i dev/debug-läge.
  
- **T2. Deterministiska seed för test**  
  Tillåt `generateByKey(key, seed?)` för reproducerbar testning utan att ändra default-beteende.

- **T3. Katalog-driven generator**  
  Flytta `PROTOCOLS`/`Difficulty`-mappningen till `lib/topology/index.ts` som enkel dispatch, utan hårdkodade tuples i `scenarios.ts`.

### 3.2 Progress & användarupplevelse
- **P1. Versionerad progress-struktur**  
  ```ts
  interface AppProgress {
    version: 1;
    labsCompleted: Record<string, boolean>;
    quizResults: Record<string, { score: number; passed: boolean; ts: number }>;
  }
  ```
  - `lib/progress.ts`: `load()`, `save()`, migrering från `v0` (ingen version) till `v1`.
  - Lägg till `resetProgress(version?: number)`.

- **P2. Enhetlig svårighetsnorm**  
  ```ts
  type Difficulty = 'Lätt' | 'Medel' | 'Svår'; // topologi
  type QuizDifficulty = 'easy' | 'medium' | 'hard'; // quiz (kan mappas)
  ```
  - Inför en mappning `const DIFFICULTY_MAP = { easy: 'Lätt', medium: 'Medel', hard: 'Svår' } as const`.
  - Använd överallt där UI renderar svårighetsgrad.

### 3.3 UI & a11y (designsystemet)
- **U1. Primitiver**  
  ```ts
  // components/primitives/
  Button.tsx   => <button> med aria-*, focus-*, disabled, variants
  Card.tsx     => <article> med rubrik, body, actions
  Badge.tsx    => <span> med color tokens för difficulty/protocol
  ```
  - Följ designsystem: CSS-variabler, inga hårdkodade färger.
  - Alla komponenter ska ha `className`-prop för override.

- **U2. Tillgänglighet**  
  - Alla knappar har tydligt `type="button"|"submit"`.
  - SVG-topolin har `role="img"` + `<title>`/`<desc>`.
  - Fokusindikator (outline) synlig för tangentbordsnavigering.
  - Färgkontrast 4.5:1 för text.

- **U3. Testning**  
  - E2E-test (Playwright eller Vitest + @testing-library/react) för att:
    - Generera topologi via katalog, verifiera länkar/CSS.
    - Klicka "Generera ny subnät" och verifiera att inga IP-bildar krockar.
    - Spara/ladda progress i localStorage.

## 4. Arbetsordning (fas-intern)

### Fas 0 — Kollaps och kontroll (1 dag)
1. Verifiera att `tsc --noEmit` är grönt.
2. Verifiera att `npm run build` är grönt.
3. Verifiera `/topologier` fungerar i dev: öppna, klicka på 3 olika labbar, generera nya subnät 3 gånger.

### Fas 1 — Buggar (2–3 dagar)
1. **Topologi:**  
   - Åtgärda B1–B8 med hjälp av en agent per generator (parallellt).  
   - Varje kör validator (T1) innan den anses fixad.
2. **Progress:**  
   - B9–B10: versionsnyckel + migrationslogik.
3. **Bygg:**  
   - B11–B12: `tsconfig.json`-rensning.

### Fas 2 — Förbättringar (2–4 dagar)
1. T1–T3: validator + determinism + generator-refaktor.
2. P1–P2: versionerad progress + difficulty-mappning.
3. U1–U3: primitiver + a11y + tester.

### Fas 3 — Gräns (Innan login/nya features)
- **Måste vara klart innan:**
  - `tsc --noEmit` → 0 fel
  - `npm run build` → 0 fel
  - Alla 10 generatorer passerar `validate()` 
  - localStorage-versionerad
  - UI-primitiver i `/components/primitives`
  - Minst 1 automatiskt test (vitest) för topologistabilitet

- **Får vänta:**
  - `/login`, `/register`, auth (t.ex. NextAuth)
  - Notifieringar (email/SMS)
  - Sociala funktioner (leaderboard)
  - Avancerad analytics
  - PWA/offline-läge

## 5. Agent-uppläpp för imorgon

| Agent | Fokus | Input | Output | Beroende |
|-------|-------|-------|--------|----------|
| A | Topologi-validator + buggfix | `scenarios.ts`, `layout.ts` | `lib/topology/validate.ts`, patchade generatorer | Ingen |
| B | localStorage-versionering | `lib/progress.ts` | Versionerad `AppProgress` + migrationslogik | Ingen |
| C | UI-primitiver + a11y | `components/` | `components/primitives/*` | Ingen |
| D | `tsconfig` + byggstäd | `tsconfig.json`, `package.json` | Grönt `tsc --noEmit` + `npm run build` | A, B, C |

- **Agent D** körs sist och verifierar hela kedjan.
- Om D misslyckas: dela upp felet och tilldela en ny agent åt feltyp (typfel → A, byggfel → C).
